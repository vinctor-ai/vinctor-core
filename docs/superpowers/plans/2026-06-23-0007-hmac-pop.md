off (token without `pop_secret` ignores any proof → byte-for-byte unchanged); gate
strictly on `token.pop_secret is not None`; **length-prefixed canonical** (no

an **injectable per-process instance** (record only AFTER mac+freshness pass;

`subject_token_invalid` 403 (no leak); `pop_secret` never leaves the mint 201 body

  pop_secret, action, resource, ts, nonce, token_id)`:
  - valid proof (fresh ts, correct mac, matching action/resource) → permit,
    `subject_token_verified=true`.
  - pop-required token + **no/blank** proof → 403 `subject_token_invalid`.
  - wrong-secret mac → 403. stale ts (`now-ts > skew`) and far-future ts → 403.
  - proof bound to a different action OR resource than the request → 403.
  - **replay:** same `(token_id, nonce)` twice → 2nd is 403; retry with a NEW
    nonce (fresh ts) → permit.
  - malformed proof (2 parts; non-int ts) → 403.
  - **non-pop token** + a present proof header → ignored, behaves as today.
  - **canonical injection:** a `\n` inside `resource` does not let a proof for one
    `(action,resource)` validate for a different split (length-prefix locks it).
  - **cache full of fresh entries → reject** (construct a `PopReplayCache(max_entries=1)`,
    record one fresh nonce, a second distinct fresh nonce → rejected, not allowed).

```python
from __future__ import annotations
import base64, hmac, hashlib
from datetime import UTC, datetime

def pop_canonical(action: str, resource: str, ts: int, nonce: str, token_id: str) -> bytes:
    parts = [action, resource, str(ts), nonce, token_id]
    out = bytearray()
    for p in parts:
        b = p.encode("utf-8")
        out += len(b).to_bytes(4, "big") + b
    return bytes(out)

def pop_mac(pop_secret: str, canonical: bytes) -> str:
    digest = hmac.new(pop_secret.encode("utf-8"), canonical, hashlib.sha256).digest()
    return base64.urlsafe_b64encode(digest).rstrip(b"=").decode("ascii")

class PopReplayCache:
    """Per-process anti-replay for PoP nonces. Single-node only (see spec Risks)."""
    def __init__(self, max_entries: int = 10000) -> None:
        self._seen: dict[tuple[str, str], int] = {}
        self._max = max_entries

    def check_and_record(self, *, token_id: str, nonce: str, ts: int, now_unix: int, skew: int) -> bool:
        cutoff = now_unix - skew
        if self._seen:
            for k in [k for k, t in self._seen.items() if t < cutoff]:
                del self._seen[k]
        key = (token_id, nonce)
        if key in self._seen:
            return False                       # replay
        if len(self._seen) >= self._max:
            return False                       # full of fresh entries -> fail closed
        self._seen[key] = ts
        return True

def verify_pop(*, proof: str | None, pop_secret: str, token_id: str, action: str,
               resource: str, now: datetime, skew: int, replay_cache: "PopReplayCache") -> bool:
    if proof is None or proof.strip() == "":
        return False
    parts = proof.split(".")
    if len(parts) != 3:
        return False
    ts_raw, nonce, mac = parts
    try:
        ts = int(ts_raw)
    except ValueError:
        return False
    if not nonce or not mac:
        return False
    if now.tzinfo is None:
        now = now.replace(tzinfo=UTC)
    now_unix = int(now.timestamp())
    if abs(now_unix - ts) > skew:
        return False
    expected = pop_mac(pop_secret, pop_canonical(action, resource, ts, nonce, token_id))
    if not hmac.compare_digest(expected, mac):
        return False
    # record ONLY after mac+freshness pass
    return replay_cache.check_and_record(token_id=token_id, nonce=nonce, ts=ts,
                                         now_unix=now_unix, skew=skew)
```

## Notes
- Length-prefixed canonical makes the binding injection-proof regardless of field
  content or validation order — that is why no pre-MAC field validation is needed.
- Record the nonce ONLY after mac+freshness pass (a bad/stale proof must not
  consume a cache slot — else it's an eviction-DoS).
- Single-process anti-replay only; document the 2×skew multi-process residual + the
  shared-cache follow-up. Default skew 30s.
- `pop_secret` is plaintext at rest (per-token, short-lived, raw token also
  required, revocable); never emitted except the one-time mint body.
